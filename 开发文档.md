# Mini Agent 开发文档

> 面向开发者的项目说明。日常使用与启动见 [操作文档](./操作文档)；Cursor Agent 路线图见 [AGENTS.md](./AGENTS.md)。

---

## 1. 项目简介

**Mini Agent** 是一个基于 FastAPI + Vue 3 的 AI Agent 聊天平台，支持：

- 多厂商大模型（DeepSeek / Qwen / Kimi / GLM）
- SSE 流式对话 + Function Calling 工具循环
- 三档思考模式（普通 / 思考 / 深度思考）
- SQLite 会话持久化
- RAG 知识库（TF-IDF 默认，可扩展 embedding）
- 文件上传与工作区工具

---

## 5. 核心模块

### 5.1 配置 `app/config.py`

- 始终从 **项目根目录** 加载 `.env`（不依赖 cwd）
- `provider_credentials(provider)` 按厂商读取独立 Key / Base URL
- 每个厂商必须使用独立变量名，**不能共用 `API_KEY`**

```python
from app.config import env, provider_credentials, default_model

key, url = provider_credentials("kimi")
model = default_model()  # 读 MODEL 或 DEFAULT_MODEL
```

### 5.2 模型路由 `app/router.py`

- 启动时注册所有模型到 `ModelRouter`
- Kimi 会从 API `/models` 动态拉取账号可用模型
- `OpenAICompatibleProvider` 懒加载凭证，改 `.env` 后重启即生效
- Kimi K2 系列自动使用 `temperature=1`

**新增模型步骤：**

1. 在 `router.py` 的 `_register_all()` 中 `_add(model_id, provider_name, api_model_name, display_name)`
2. 在 `.env` 配置对应 `*_API_KEY` 和 `*_BASE_URL`
3. 前端 `ModelSelector.vue` 分组规则会自动匹配（或补充 `match` 函数）

### 5.3 Provider `app/providers/openai_compat.py`

统一 OpenAI 兼容接口，SSE 解析 **强制 UTF-8**（修复 Windows 中文乱码）：

```python
# Provider 层事件
{"type": "content", "delta": "..."}      # 文本 token
{"type": "tool_calls", "tool_calls": [...], "content": "..."}
{"type": "finish", "content": "..."}     # 无工具，流结束
```

### 5.4 Agent `app/agent.py`

```python
def run_stream(self, task: str, mode: str = "normal"):
```

**工具循环逻辑：**

1. 构建 messages（system prompt + 历史 + 文件摘要）
2. 调用 `provider.chat_stream(messages, tools=...)`
3. 若返回 `tool_calls` → 执行工具 → append tool result → 继续循环
4. 若无工具 → 保存 assistant 消息 → yield `done`

**Agent 层事件（→ SSE → 前端）：**

| 事件 | 含义 |
|------|------|
| `token` | 文本片段，追加到消息 |
| `thinking` | 原生思考链片段（`reasoning_content`） |
| `status` | 工具调用状态，如「🔧 正在调用 calculator...」 |
| `reset` | 清空当前流式内容（工具轮次切换，仅在有已输出内容时触发） |
| `done` | 完成，含 `content` / `thinking` / `title` / `model` |
| `error` | 错误信息 |

### 5.5 原生思考 `app/thinking.py`

单一布尔参数 **`thinking`** 控制是否开启原生思考（API 级，非 prompt 模拟）：

| 开关 | 参数 | 说明 |
|------|------|------|
| 关 | `thinking=false` | 普通回答 |
| 开 | `thinking=true` | 模型返回 `reasoning_content` 思考链 |

**各厂商 API 参数：**

| 厂商 | 开启 | 关闭 |
|------|------|------|
| DeepSeek | `thinking.type=enabled` | `thinking.type=disabled` |
| Qwen / Kimi / GLM | `enable_thinking=true` | `enable_thinking=false` |

**能力检测** `supports_native_thinking(provider, model)`：DeepSeek 全系列；Qwen 全系列；Kimi 仅 K2/thinking 模型；GLM 仅 glm-4 系列。

**兼容旧接口**：`mode=think/deep` 会映射为 `thinking=true`；`GET /modes` 保留但已废弃。

**前端**：`ThinkingToggle.vue` 开关 → `POST /chat/stream` 传 `thinking=true/false`；`MessageItem.vue` 折叠展示思考过程。

### 5.6 工具系统 `app/tools/`

**新增工具：**

```python
# app/tools/my_tool.py
from app.tools.base import Tool

class MyTool(Tool):
    name = "my_tool"
    description = "工具说明"
    parameters = {
        "type": "object",
        "properties": {
            "query": {"type": "string", "description": "..."},
        },
        "required": ["query"],
    }

    def execute(self, args: dict) -> str:
        return f"结果: {args.get('query')}"
```

```python
# app/tools/__init__.py
from app.tools.my_tool import MyTool
reg.register(MyTool())
```

重启后端后 Agent 的 system prompt 会自动包含新工具。

**已注册工具：**

| 工具 | 说明 |
|------|------|
| `list_files` / `search_file` / `read_file` / `write_file` | workspace 文件操作 |
| `calculator` | AST 安全数学计算 |
| `db_query` | 会话统计 |
| `rag_search` / `rag_index` / `rag_status` | RAG 知识库 |
| `web_search` | 预留（未接入真实搜索） |

### 5.7 会话与持久化

- **运行时**：`SessionManager.sessions` 字典持有 `Agent` 实例
- **持久化**：SQLite `data/mini_agent.db`
- 对话完成后 `sessions.save(session_id)` 写入 DB
- 首次启动自动从 `data/sessions.json` 迁移（备份为 `.bak`）

**表结构：**

| 表 | 字段 |
|----|------|
| `sessions` | id, title, model, created_at, updated_at |
| `messages` | session_id, role, content, created_at |
| `files` | session_id, filename, content, created_at |

> 思考模式 `mode` 目前仅存于前端会话状态，未持久化到 DB。如需持久化，在 `sessions` 表加 `mode` 字段并同步 API。

### 5.8 RAG `app/rag/`

```
上传/索引 → chunker → embedder → store → retriever → rag_search 工具
```

- 默认 `RAG_EMBEDDING_MODE=tfidf`，无需下载模型
- 上传文件时 `server.py` 自动调用 `rag_service.index_document()`
- 数据目录：`data/rag/`

## 6. SSE 全链路

```
用户发送消息
  → App.vue 构建 FormData (message, session_id, model, mode, file?)
  → POST /chat/stream
  → SessionManager.get_agent()
  → agent.run_stream(task, mode)
  → provider.chat_stream()  ← 各厂商 API
  → agent 转换事件类型
  → server._sse_event()     ← JSON UTF-8 bytes
  → 前端 streamChat()       ← TextDecoder('utf-8')
  → onToken / onStatus / onReset / onDone
  → MessageItem 渲染
```

**开发注意：**

- 开发环境 SSE **直连** `http://127.0.0.1:8000/chat/stream`（绕过 Vite 代理缓冲）
- 其他 API 走 `/api` 代理
- 事件类型改动需同步：`openai_compat.py` → `agent.py` → `server.py` → `api/chat.js`

## 7. API 参考

### 7.1 健康与元数据

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/health` | 连接状态、默认模型、env 路径 |
| GET | `/models` | 模型列表 + 各厂商 configured 状态 |
| GET | `/modes` | 思考模式列表 |
| GET | `/tools` | 已注册工具 |

### 7.2 会话

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/sessions` | 会话列表 |
| POST | `/sessions` | 创建（form: session_id, title, model） |
| GET | `/sessions/{id}` | 会话详情 + 消息 |
| PUT | `/sessions/{id}/model` | 切换模型（form: model） |
| DELETE | `/sessions/{id}` | 删除 |

### 7.3 对话

**POST `/chat/stream`**（主接口，SSE）

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 否* | 用户消息 |
| session_id | string | 是 | 会话 ID |
| model | string | 否 | 模型 ID，不传则用会话/默认 |
| mode | string | 否 | normal / think / deep，默认 normal |
| file | file | 否* | 上传文件 |

\* message 与 file 至少一个

**SSE 响应示例：**

```
data: {"type": "token", "content": "你好"}
data: {"type": "status", "content": "🔧 正在调用 calculator..."}
data: {"type": "reset"}
data: {"type": "done", "content": "...", "title": "...", "model": "kimi", "mode": "deep"}
```

**POST `/chat`** — 非流式兼容接口，参数同上，返回 `{"answer": "...", "model": "..."}`

### 7.4 RAG

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/rag/status` | 索引状态 |
| GET | `/rag/search?q=` | 调试检索 |

### 7.5 文件

| 方法 | 路径 | 说明 |

## 8. 环境变量

`.env` 位于项目根目录，完整示例：

```env
MODEL=deepseek-v4-flash

# DeepSeek
DEEPSEEK_API_KEY=sk-...
DEEPSEEK_BASE_URL=https://api.deepseek.com/v1

# Qwen（必须用 DashScope 兼容地址）
QWEN_API_KEY=sk-...
QWEN_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1

# Kimi（国内 Key 用 moonshot.cn，不是 moonshot.ai）
KIMI_API_KEY=sk-...
KIMI_BASE_URL=https://api.moonshot.cn/v1
KIMI_MODEL=moonshot-v1-32k

# GLM
GLM_API_KEY=...
```

**规则：**

- 每行 `KEY=value`，不加引号

## 11. 开发约定

1. **SSE 事件类型**不可随意改名，改动需全链路同步
2. **新增工具**走 `ToolRegistry.register()`，不要写死 handler dict
3. **新增模型**实现 `BaseModelProvider` 或在 `router.py` 注册 OpenAI 兼容模型
4. **会话 ID** 前端用 `String(Date.now())` 字符串
5. **中文编码** Provider SSE 必须 UTF-8 解析；服务端 SSE 输出 UTF-8 bytes
6. **不要用** `app/main.py`（旧 mock 入口）
7. **配置加载** 用 `app.config.env()`，不要直接 `os.getenv`（会漏掉根目录 .env）

## 13. MCP 集成

Mini Agent 同时支持 **MCP Server**（对外暴露工具）和 **MCP Client**（接入外部 MCP 工具）。

### 13.1 MCP Server — 给 Cursor 用

将本项目的 Agent 工具暴露为标准 MCP 服务，Cursor / Claude Desktop 可直接调用。

**启动（stdio）：**

```powershell
cd F:\mini-agent
.\venv\Scripts\python.exe -m app.mcp.server
```

**Cursor 配置**（已生成 `.cursor/mcp.json`）：

```json
{
  "mcpServers": {
    "mini-agent": {
      "command": "F:\\mini-agent\\venv\\Scripts\\python.exe",
      "args": ["-m", "app.mcp.server"],
      "cwd": "F:\\mini-agent"
    }
  }
}
```

配置后重启 Cursor，可在 MCP 面板看到 `calculator`、`read_file`、`rag_search` 等 9 个工具。

**代码位置：** `app/mcp/server.py`

### 13.2 MCP Client — Agent 调用外部 MCP

让聊天 Agent 自动使用外部 MCP Server 的工具（如官方 filesystem server）。

**步骤：**

1. 复制配置示例：
   ```powershell
   copy mcp_servers.json.example mcp_servers.json
   ```

2. 编辑 `mcp_servers.json`：
   ```json
   [
     {
       "name": "filesystem",
       "transport": "stdio",
       "command": "npx",
       "args": ["-y", "@modelcontextprotocol/server-filesystem", "F:/mini-agent/workspace"]
     }
   ]
   ```

3. 启用 MCP：
   ```env
   MCP_ENABLED=true
   ```

4. 重启后端。外部工具会以 `mcp_{server}__{tool}` 名称注册到 ToolRegistry。

**API：**

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/mcp/status` | MCP 连接状态与已加载工具 |
| POST | `/mcp/reload` | 重新连接 MCP 服务器 |

**代码位置：**

| 文件 | 职责 |
|------|------|
| `app/mcp/config.py` | 读取 `mcp_servers.json` / `MCP_SERVERS` |
| `app/mcp/bridge.py` | 后台线程维持 MCP 连接 |
| `app/mcp/tool_adapter.py` | 将 MCP 工具注入 ToolRegistry |

### 13.3 传输类型

| transport | 配置字段 |
|-----------|----------|
| `stdio` | `command`, `args`, `env?` |
| `sse` | `url`, `headers?` |

### 13.4 依赖

```powershell
pip install "mcp[cli]"
```

已写入 `requirements.txt`。
